# 快速回归测试用例（冒烟测试）

## 说明

每次部署或大版本发布前，运行以下冒烟测试确认核心功能正常。预计执行时间：5-10 分钟。

## 前置条件

- 前后端已启动
- MySQL 已运行，种子数据已初始化
- 准备一个可用浏览器

---

## TC-S-01: 首页加载

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 访问 `http://localhost:5173` |
| 预期 | 页面正常加载，显示标题与导航栏 |
| 耗时 | ~30s |

## TC-S-02: 注册新用户

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 点击"免费注册"<br>2. 填写姓名/邮箱/密码<br>3. 点击注册 |
| 预期 | 注册成功，跳转到用户仪表盘 |
| 耗时 | ~30s |

## TC-S-03: 用户登录

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 点击"登录"<br>2. 输入刚注册的邮箱和密码<br>3. 点击登录 |
| 预期 | 登录成功 |
| 耗时 | ~20s |

## TC-S-04: 预约流程

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 点击"预约咨询"<br>2. 选择张医生<br>3. 选择日期<br>4. 选择 09:00<br>5. 点击确认预约 |
| 预期 | 预约创建成功，状态"待确认" |
| 耗时 | ~60s |

## TC-S-05: 管理员登录并查看仪表盘

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 登出当前用户<br>2. 登录 admin@test.com / password<br>3. 查看仪表盘 |
| 预期 | 管理员登录成功，仪表盘显示 5 个统计卡片 |
| 耗时 | ~30s |

## TC-S-06: 管理员管理用户

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 点击"用户管理"<br>2. 找到刚注册的用户<br>3. 点击"禁用" |
| 预期 | 用户状态变为"已停用" |
| 耗时 | ~20s |

## TC-S-07: 管理员管理网点

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 点击"网点管理"<br>2. 新增一个网点<br>3. 删除刚新增的网点 |
| 预期 | 新增和删除均成功 |
| 耗时 | ~30s |

## TC-S-08: 未登录访问后台

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 登出<br>2. 直接访问 `/admin/dashboard` |
| 预期 | 被重定向到登录页 |
| 耗时 | ~10s |

---

## 通过标准

所有 8 个用例全部通过 → 冒烟测试通过。

任一用例失败 → 检查对应模块，修复后重新运行冒烟测试。
