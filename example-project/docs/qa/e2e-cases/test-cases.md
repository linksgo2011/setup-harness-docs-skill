# E2E Test Cases — 咨询预约平台

## 1. Public Pages

| # | 用例 | 步骤 | 预期 |
|---|---|---|---|
| 1.1 | 首页加载 | 访问 `/` | 页面显示标题"专业咨询服务"、"免费注册"、"立即预约"按钮、版权信息 |
| 1.2 | 登录页渲染 | 访问 `/login` | 显示邮箱/密码输入框、登录按钮、注册链接 |
| 1.3 | 注册页渲染 | 访问 `/register` | 显示姓名/邮箱/手机/密码输入框、注册按钮、登录链接 |
| 1.4 | 公开 API 可用 | GET `/api/v1/data-dictionaries`、`/api/v1/business-rules`、`/api/v1/consultants` | 均返回 200，咨询师列表 ≥ 3 条 |
| 1.5 | 导航跳转 | 首页点击"登录" → 点击"立即注册" | URL 依次变为 `/login` → `/register` |

## 2. Authentication

| # | 用例 | 步骤 | 预期 |
|---|---|---|---|
| 2.1 | 管理员登录成功 | 填写 admin@test.com / password → 提交 | 跳转到 `/admin/dashboard`，显示"管理仪表盘" |
| 2.2 | 错误密码提示 | 填写 admin@test.com / wrongpassword → 提交 | 显示错误提示 |
| 2.3 | 注册新用户 | 填写姓名/邮箱/手机/密码 → 提交 | 跳转到 `/user/dashboard`，显示用户姓名 |
| 2.4 | 重复邮箱注册 | 使用已注册邮箱再次注册 | 显示错误提示 |
| 2.5 | 新用户登录 | 使用刚注册的账号登录 | 跳转到 `/user/dashboard`，显示"用户中心" |

## 3. User Flows

| # | 用例 | 步骤 | 预期 |
|---|---|---|---|
| 3.1 | 用户仪表盘 | 登录后访问 `/user/dashboard` | 显示"总预约"、"待确认"、"已完成"统计卡片 |
| 3.2 | 浏览咨询师 | 访问 `/user/book` | 显示张医生、李律师、王教练三条记录 |
| 3.3 | 预约 + 取消 | 选咨询师 → 选日期 → 选时间 → 填备注 → 提交 → 取消 | 预约成功跳转预约列表 → 取消后显示"已取消"状态 |
| 3.4 | 编辑个人信息 | 访问 `/user/profile` → 修改姓名 → 保存 | 弹出"保存成功"提示 |

## 4. Admin Flows

| # | 用例 | 步骤 | 预期 |
|---|---|---|---|
| 4.1 | 仪表盘统计 | 管理员登录 → 访问 `/admin/dashboard` | 显示用户数 / 咨询师 / 总预约 / 待确认 / 网点五个卡片 |
| 4.2 | 用户管理 | 访问 `/admin/users` | 表格显示 admin@test.com、管理员；可点击禁用/启用 |
| 4.3 | 咨询师管理 | 侧栏点击"咨询师管理" | 表格显示张医生、李律师、王教练 |
| 4.4 | 预约管理 | 侧栏点击"预约管理" | 页面标题为"预约管理" |
| 4.5 | 网点 CRUD | 新增网点(名称/地址/电话) → 保存 → 编辑 → 保存 → 删除 | 表格显示新建条目 → 修改后名称更新 → 删除后消失 |

## 5. Edge Cases

| # | 用例 | 步骤 | 预期 |
|---|---|---|---|
| 5.1 | 未登录访问管理页 | 清除 localStorage → 访问 `/admin/dashboard` | 跳转到 `/login` |
| 5.2 | 未登录访问用户页 | 清除 localStorage → 访问 `/user/dashboard` | 跳转到 `/login` |
| 5.3 | 首页 CTA 可见 | 未登录状态下访问 `/` | "免费注册"按钮可见 |
