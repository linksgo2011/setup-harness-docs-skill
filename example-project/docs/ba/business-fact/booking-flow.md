# 预约业务流程事实

## 1. 角色定义

| 角色 | 说明 | 系统表示 |
|------|------|----------|
| 普通用户（访客） | 未登录，可浏览首页与咨询师列表 | 无认证 / ROLE_USER |
| 注册用户 | 已登录，可预约、查看/取消自己的预约、编辑资料 | ROLE_USER |
| 管理员 | 已登录，可查看仪表盘统计、管理用户/咨询师/预约/网点 | ROLE_ADMIN |

## 2. 预约流程（4 步）

```
Step 1: 选择咨询师
  - 系统展示所有 ACTIVE 状态的咨询师，按 sortOrder 升序
  - 用户点击选中一个

Step 2: 选择日期
  - 系统展示未来 7 天（含今日）的日期按钮
  - 日期连续，无间隔
  - 用户点击选中一个

Step 3: 选择时间
  - 系统根据选中的咨询师 + 日期，查询可用时段
  - 可用时段 = 09:00-18:00 内，每小时一个 slot（09:00, 10:00, ..., 17:00）
  - 排除已存在非取消状态的预约
  - 用户点击选中一个

Step 4: 确认预约
  - 用户可填写备注（选填）
  - 提交后系统创建预约，状态为 PENDING（待确认）
  - 系统检查同一咨询师、同一日期、同一时段是否已有非取消预约 → 冲突则拒绝（409）
```

## 3. 预约状态机

```
PENDING ──► CONFIRMED ──► COMPLETED
  │              │
  └──────┬───────┘
         ▼
      CANCELLED
```

| 状态 | 含义 | 可操作 |
|------|------|--------|
| PENDING | 待确认（用户已提交，管理员未确认） | 取消 |
| CONFIRMED | 已确认（管理员已确认） | 取消、完成 |
| COMPLETED | 已完成（终态） | 无 |
| CANCELLED | 已取消（终态） | 无 |

注：从 PENDING 和 CONFIRMED 可取消，取消时需要填写原因。

## 4. 业务规则

### 预约相关

| 规则 | 描述 | 实施方式 |
|------|------|----------|
| 冲突检测 | 同一咨询师同一时段不可重复预约 | `uk_appointment_slot` 唯一索引 + `AppointmentConflictService` |
| 时段间隔 | 预约时段间隔为 60 分钟 | 前端 time slot 按 1h 步进，后端 slot.intervalMinutes = 60 |
| 可用时段 | 可预约时段为 09:00-18:00 | 前端展示 09:00-17:00，后端 slot.available.start/end |
| 取消窗口 | PENDING 和 CONFIRMED 状态可取消 | `Appointment.canCancel()` 守卫方法 |

### 认证相关

| 规则 | 描述 | 实施方式 |
|------|------|----------|
| 密码最小长度 | 密码最少 8 位 | 前端 + 后端校验 |
| JWT 有效期 | JWT 有效期 24 小时 | `jwt.expiration=86400000` |
| 密码算法 | 密码使用 BCrypt 加密 | `PasswordService` → `BCryptPasswordEncoder` |

## 5. 前台 vs 后台功能划分

### 前台（/user/*，需 ROLE_USER）

| 页面 | 功能 |
|------|------|
| 仪表盘 | 展示本人预约统计：总预约数、待确认数、已完成数 |
| 预约管理 | 列出本人预约，可取消 PENDING/CONFIRMED 状态的预约 |
| 预约咨询 | 4 步预约流程 |
| 个人中心 | 编辑姓名、手机号；修改密码 |

### 后台（/admin/*，需 ROLE_ADMIN）

| 页面 | 功能 |
|------|------|
| 仪表盘 | 全局统计：用户数、咨询师数、总预约数、待确认数、网点数 |
| 用户管理 | 列出所有用户，可启用/禁用用户 |
| 咨询师管理 | 列出所有咨询师 |
| 预约管理 | 列出所有预约 |
| 网点管理 | CRUD 网点（新增、编辑、删除） |
